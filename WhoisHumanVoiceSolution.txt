Build a minimal, reliable local text-to-speech prototype using Kokoro TTS.

The immediate goal is to prove this pipeline:

Browser text input
→ local FastAPI server
→ Kokoro speech generation
→ WAV file
→ browser playback

This is a technical spike only. Do not integrate Unity, AI agents, game logic, speech recognition, lip sync, or cloud APIs yet.

## Core requirements

Create a self-contained project named:

```text
kokoro-local-tts
```

Use:

* Python 3.11 or 3.12
* FastAPI
* Uvicorn
* Official `kokoro` Python package
* `soundfile`
* `numpy`
* Plain HTML, CSS, and JavaScript
* No React
* No paid APIs
* No Docker unless the normal local setup is already working

Use the official Kokoro implementation and documentation:

* https://github.com/hexgrad/kokoro
* https://huggingface.co/hexgrad/Kokoro-82M
* https://huggingface.co/hexgrad/Kokoro-82M/blob/main/VOICES.md

Do not use an unofficial wrapper unless the official package cannot run after reasonable debugging. If a fallback is required, document the reason before changing implementations.

## Kokoro service

Create a `KokoroTtsService` class.

Requirements:

1. Initialize `KPipeline` only once.
2. Do not reload the model for every request.
3. Default to American English:

   ```python
   KPipeline(lang_code="a")
   ```
4. Default voice:

   ```text
   af_heart
   ```
5. Default speed:

   ```text
   1.0
   ```
6. Kokoro may return multiple audio segments. Concatenate all generated segments into one NumPy array.
7. Write WAV audio at 24,000 Hz.
8. Protect generation with a lock if concurrent inference is unsafe.
9. Handle empty generation cleanly.
10. Log generation duration, character count, selected voice, and cache status.
11. Never log secret data because this service should not require any secrets.

Do not initialize a new pipeline inside the request handler.

Do not hard-code voice names without checking that they work in the installed version.

Create a curated initial list of at least six working English voices:

* at least three female voices;

Also create `scripts/list_voices.py` to inspect or test available voices.

## API endpoints

### Health

```text
GET /api/health
```

Return:

```json
{
  "status": "ok",
  "model": "Kokoro-82M",
  "sampleRate": 24000,
  "device": "cpu",
  "pipelineLoaded": true
}
```

Return a meaningful error state if model initialization failed.

### List voices

```text
GET /api/voices
```

### Generate speech

```text
POST /api/tts
```

Request:

```json
{
  "text": "I believe Agent Four contradicted their previous statement.",
  "voice": "af_heart",
  "speed": 1.0
}
```

Validation:

* `text`: 1–500 characters
* `voice`: must be in the supported voice list
* `speed`: 0.7–1.3
* trim surrounding whitespace
* reject unsupported values with clear HTTP 400 responses

Response:

```json
{
  "audioId": "sha256-based-id",
  "audioUrl": "/api/audio/sha256-based-id.wav",
  "voice": "af_heart",
  "speed": 1.0,
  "cached": false
}
```

### Retrieve generated WAV

```text
GET /api/audio/{audio_id}.wav
```

Return the file with:

```text
Content-Type: audio/wav
```

Use FastAPI `FileResponse`.

Prevent path traversal. The client must never supply an arbitrary file path.

## Audio caching

Use a deterministic SHA-256 cache key generated from:

```text
text + voice + speed + model version
```

Store WAV files in:

```text
generated_audio/
```

If the same request is sent again:

* do not regenerate it;
* return the existing file;
* set `cached: true`.

Do not create a database.

## CORS and security

Because FastAPI also serves the test page, CORS should not normally be required.

If development CORS is added:

* allow only localhost origins;
* do not use `allow_origins=["*"]` unnecessarily;
* bind Uvicorn to `127.0.0.1` by default;
* do not expose the service to the public network.

## Error handling

Handle and display:

* Kokoro package missing;
* model download failure;
* voice unavailable;
* `espeak-ng` missing;
* empty text;
* invalid speed;
* synthesis failure;
* WAV write failure;
* browser fetch failure.

The server must return structured JSON errors:

```json
{
  "error": {
    "code": "TTS_GENERATION_FAILED",
    "message": "Human-readable explanation"
  }
}
```

Do not return Python stack traces to the browser.

Log full exceptions only in the terminal.
